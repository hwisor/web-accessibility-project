Accessibility Benefits Everyone

There's a misconception that design must be sacrificed in order to have an accessible website. Many sites are developed without thinking of accessibility as part of the process, making them difficult or impossible to use. But the same principles that make a website accessible also benefit all users and businesses, not just people with disabilities.

Accessibility is essential for designers that want to create visually-appealing websites, but also provide equal access to everyone.

Design Considerations

Semantics and Structure

Semantic markup uses HTML tags properly and in the correct DOM order to be programmatically determined.

Alternative Text

Non-text content has a text alternative. This includes captions and transcripts for multimedia content.

Keyboard Navigation

The functionality of content is available using only the keyboard and focusable elements have tab focus.

Programmatic Labels

Custom controls have the correct name/label, role, and value (either with HTML or with ARIA).

Color and Contrast

Elements have sufficient contrast and color is not the only visual means of conveying information.

Testing

Using automated tools and assistive technology, like screen readers, to check for accessibility.

Further Information

Website accessibility is a legal matter in many countries, with accessibility non-compliance amounting to discrimination. Web Content Accessibility Guidelines (WCAG) 2 define the principles, guidelines, success criteria, and techniques needed for accessibility. Review their detailed methodology to see if your website adheres to accessibility standards.
